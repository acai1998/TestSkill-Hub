# Bug 单字段规范

## 必填字段
以下字段必须填写，不得为空：

| 字段 | 格式要求 |
| --- | --- |
| 标题 | `[模块名] 现象描述（触发条件）`，不超过 50 字，使用中文 |
| 严重等级 | P0/P1/P2/P3，必须从定义中选择 |
| 优先级 | 高/中/低 |
| 缺陷类型 | 功能/性能/UI/数据/安全/兼容性 |
| 复现步骤 | 必须包含前置条件，步骤从 1 开始编号 |
| 预期结果 | 必须描述正确行为，可观察可验证 |
| 实际结果 | 必须描述实际异常，与预期形成对比 |

## 选填字段
| 字段 | 说明 |
| --- | --- |
| 发现版本 | App 版本号或接口版本 |
| 测试环境 | 预发/测试/线上 |
| 操作系统 | iOS 版本/Android 版本/浏览器版本 |
| 测试账号 | 账号类型（普通用户/商家/管理员等） |
| 截图/录屏 | 文件路径或描述 |
| 日志摘要 | 关键错误信息，不超过 20 行 |

## 标题写法规范
✅ 好的标题：
- `[支付] 微信支付成功后订单状态仍显示"待支付"（iOS 17）`
- `[登录] 手机号登录时验证码倒计时结束后按钮不可点击`

❌ 不好的标题：
- `支付有问题` — 太模糊
- `Bug：登录页面` — 缺少现象描述
- `测试发现一个问题，用户登录后点击支付进入支付页面后选择微信支付提交订单后...` — 太长

## 复现步骤规范
```
前置条件：已登录账号 xxx，购物车中有商品 xxx
1. 打开 App，进入购物车页面
2. 勾选所有商品，点击「结算」
3. 选择「微信支付」，点击「立即支付」
4. 在微信中完成支付
5. 返回 App 查看订单状态
```
