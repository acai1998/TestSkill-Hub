## 需求追踪与待确认项

输出中必须包含“覆盖与追踪”与“待确认项”两部分。

**错误（没有需求映射，也没有待确认项）：**

```markdown
## 三、完整测试用例
| 模块 | 测试点 | ... |
```

**正确（显式映射需求点）：**

```markdown
## 四、覆盖与追踪
| 需求点 | 覆盖说明 |
| --- | --- |
| RQ-01 下单成功 | TC-001, TC-002 覆盖主流程与库存扣减 |
| RQ-02 订单超时关闭 | TC-010 覆盖超时回收场景 |

## 五、待确认项
- [ ] RQ-03 未明确游客下单是否允许
- [ ] 缺少失败重试上限配置说明
```

### 使用原则

- 信息不完整时先基于合理假设继续生成
- 假设必须进入“待确认项”，避免隐式决策
- 每个核心需求点至少映射 1 条测试用例
