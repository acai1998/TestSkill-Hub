## 多维度覆盖

用例设计至少覆盖以下维度：
- 完整覆盖需求文档
- 功能正确性
- 边界与等价类
- 异常与容错
- 权限与安全
- 兼容性
- 性能（仅在需求包含 SLA/并发指标时）

**错误（只有功能 happy path）：**

```markdown
| 功能模块 | 测试点 |
| -------| ----- |
| 登录 | 用户名密码登录成功 |
```

**正确（覆盖维度完整）：**

```markdown
| 模块 | 测试点 | 维度 |
| --- | --- | --- |
| 登录 | 用户名密码登录成功 | 功能 |
| 登录 | 密码最小长度边界 | 边界 |
| 登录 | 密码错误重试超过上限 | 异常 |
| 登录 | 普通用户访问管理员页 | 权限/安全 |
| 登录 | iOS Safari 登录页渲染 | 兼容性 |
```
