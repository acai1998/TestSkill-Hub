# 章节

此文件定义测试用例技能的规则章节、排序、影响级别和描述。
章节 ID（括号中）是用于分组规则的文件名前缀。

---

## 1. 产物落地 (artifact)

**影响：** 关键  
**描述：** 结果必须落地为 Markdown 或 XMind 文件，禁止只在会话页面展示。

## 2. 覆盖维度 (coverage)

**影响：** 高  
**描述：** 用例必须覆盖功能、边界、异常、权限/安全、兼容性等核心维度。

## 3. 用例可执行性 (quality)

**影响：** 中高  
**描述：** 每条用例都需要可执行、可验证，步骤原子化、预期可观测。

## 4. 追踪与待确认项 (trace)

**影响：** 中等  
**描述：** 用例要与需求点建立映射，并显式输出信息缺口与待确认项。

## 5. XMind 转换 (xmind)

**影响：** 中低  
**描述：** 当需要脑图交付时，使用统一结构和转换流程产出 `.xmind` 文件。
