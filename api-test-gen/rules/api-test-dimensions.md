# 接口测试维度定义

## 核心测试维度

### 1. 正常路径（Happy Path）
- 所有必填参数完整、格式正确
- 使用有效的鉴权信息
- 期望返回 200/201，业务成功码
- 验证响应结构和关键字段值

### 2. 参数完整性
- 缺少单个必填参数（逐一测试每个必填字段）
- 所有必填参数都缺失
- 多传未定义参数（验证容错）

### 3. 参数类型校验
- 整型字段传字符串
- 字符串字段传数组/对象
- 布尔字段传非法值（如 "yes"/"no"）
- 日期字段传非法格式

### 4. 边界值测试
- 数值：最大值、最小值、最大值+1、最小值-1、0、负数
- 字符串：最大长度、最大长度+1、最小长度（1字符）、空字符串
- 数组/列表：空数组、单元素、最大元素数、超出最大数

### 5. 枚举与格式
- 传入文档未定义的枚举值
- 格式类参数：手机号（11位、非数字、少一位）、邮箱（缺@、缺域名）、日期（非法格式）

### 6. 鉴权与权限
- 无 Token/Cookie
- Token 格式非法
- Token 已过期
- 权限不足（跨用户访问、低权限角色访问高权限接口）

### 7. 幂等性（适用于 PUT/DELETE）
- 连续调用两次相同请求，验证结果一致
- 对已删除资源再次执行删除

### 8. 并发与性能（按需）
- 高并发下接口稳定性（仅在需求有 SLA 要求时）

## 优先级排序
必须覆盖：正常路径 > 参数缺失 > 鉴权
推荐覆盖：类型错误 > 边界值 > 枚举非法
按需覆盖：幂等性 > 并发
