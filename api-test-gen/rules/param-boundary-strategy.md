# 参数边界值与等价类策略

## 等价类划分原则
将参数的所有可能值划分为若干等价类，每类取一个代表值测试：
- **有效等价类**：符合规格的输入，期望正常处理
- **无效等价类**：不符合规格的输入，期望返回错误

## 数值类型边界值策略

| 约束条件 | 测试值 |
| --- | --- |
| `min <= x <= max` | min-1, min, min+1, 中间值, max-1, max, max+1 |
| `x > 0` | -1, 0, 1, 最大正整数 |
| 无约束整数 | 0, 1, -1, 最大整数, 最小整数 |
| 小数精度（2位） | 0.001（超精度）, 0.01, 正常值, 最大值 |

## 字符串类型边界值策略

| 约束条件 | 测试值 |
| --- | --- |
| `minLen <= len <= maxLen` | 空字符串, 1字符, minLen-1, minLen, maxLen, maxLen+1 |
| 格式限制（如手机号） | 正确格式, 少一位, 多一位, 含字母, 全空格 |
| 特殊字符 | SQL 注入串: `' OR 1=1`, XSS: `<script>alert(1)</script>` |
| 中英文混合 | 纯中文, 纯英文, 中英混合, Emoji |

## 数组/列表类型策略

| 场景 | 测试值 |
| --- | --- |
| 空数组 | `[]` |
| 单元素 | `[item1]` |
| 正常数量 | 文档示例数量 |
| 最大数量 | 文档限制最大值 |
| 超出最大数量 | 最大值 + 1 |
| 重复元素 | `[item1, item1]` |

## 常见格式参数测试值

### 手机号
- 正确：`13812345678`
- 少一位：`1381234567`
- 多一位：`138123456789`
- 含字母：`1381234567a`
- 空：`""`

### 日期时间
- 正确：`2026-03-23`
- 格式错误：`2026/03/23`、`20260323`
- 非法日期：`2026-13-01`、`2026-02-30`

### 金额
- 正确：`99.99`
- 负数：`-1`
- 超精度：`99.999`
- 超大值：`999999999999`
