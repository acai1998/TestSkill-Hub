# 质量等级评估标准

## 评级维度
综合以下三个维度给出质量等级：
1. 用例通过率
2. 遗留 P0/P1 Bug 数量
3. Bug 修复率

## 质量等级定义

### ⭐⭐⭐⭐ 优秀
- 用例通过率 ≥ 95%
- 遗留 P0 = 0，遗留 P1 = 0
- Bug 修复率 ≥ 95%

### ⭐⭐⭐ 良好
- 用例通过率 ≥ 90%
- 遗留 P0 = 0，遗留 P1 ≤ 2（有 Workaround）
- Bug 修复率 ≥ 85%

### ⭐⭐ 一般
- 用例通过率 ≥ 80%
- 遗留 P0 = 0，遗留 P1 ≤ 5
- Bug 修复率 ≥ 70%

### ⭐ 较差
- 用例通过率 < 80%
- 存在遗留 P0，或遗留 P1 > 5
- Bug 修复率 < 70%

## 特殊情况说明
- 即使通过率很高，只要存在遗留 P0，质量等级不能高于"一般"。
- 测试范围缩减（如 Block 导致未执行）需在报告中说明，不影响等级计算。
- 新功能首次测试周期较短时，可在备注中说明背景。
