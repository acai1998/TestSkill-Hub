# 上线建议决策规则

## 建议分类

### ✅ 建议上线
满足以下所有条件：
- 遗留 P0 Bug = 0
- 遗留 P1 Bug = 0（或全部有可接受的 Workaround）
- 用例通过率 ≥ 90%
- 核心流程（主干路径）100% 通过

### ⚠️ 条件上线
满足以下条件（需明确列出前置条件）：
- 遗留 P0 Bug = 0
- 遗留 P1 Bug ≤ 2，且均有 Workaround
- 用例通过率 ≥ 80%
- 前置条件：遗留 P1 在 XX 版本修复 / 上线后监控 XX 指标

### ❌ 建议不上线
触发以下任一条件：
- 存在遗留 P0 Bug
- 遗留 P1 Bug > 2 且无 Workaround
- 用例通过率 < 80%
- 核心主流程存在失败用例

## 上线建议说明规范
给出建议时必须包含：
1. **建议结论**：明确的建议上线/条件上线/建议不上线
2. **决策依据**：引用具体数据（如"遗留 P1 Bug 2 个，均有 Workaround"）
3. **前置条件**（条件上线时）：必须完成的事项，逐条列出
4. **风险提示**：上线后需关注的风险点和监控指标

## 注意事项
- 上线建议是参考意见，最终决策由产品/研发负责人确认。
- 若测试覆盖不完整（如接口测试未执行），需在风险提示中说明。
- 灰度发布场景下，可适当降低通过率要求，但需说明灰度策略。
